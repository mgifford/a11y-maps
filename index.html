Accessible Mapping Recipes

A collection of simplified, copy-pasteable patterns for building accessible maps.

  • Recipe 01: The Basics (Wrapper & Semantics)

    The "Hello World" of accessible mapping. Learn the minimum valid HTML structure.

    Key Learnings: <figure> wrappers, ARIA roles, and enabling keyboard panning.

    View Recipe 01

  • Recipe 02: Map-List Synchronization

    The "Golden Rule" of map accessibility: "Everything on the map must be in a list."

    Key Learnings: Bi-directional focus management, semantic list buttons, and state synchronization (`aria-current`).

    View Recipe 02

  • Recipe 03: Patterns & Alternatives

    How to design for color blindness and provide text-based alternatives for spatial questions.

    Key Learnings: SVG Patterns (hatching/dots), HTML-based Legends, and "Address Search" as an accessible alternative to "Point-in-Polygon" tasks.

    View Recipe 03

  • Recipe 04: Accessible Directions

    Maps are visual. How do blind users know where to go? We pair the map with a synchronized text list.

    Key Learnings: The "Directions List" pattern, and step-by-step focus management.

    View Recipe 04

  • Recipe 05: Filtered Map

    Too many pins cause cognitive overload and keyboard fatigue. Filters help users find what they need.

    Key Learnings: Reducing tab stops via filtering, and semantic ARIA Live Regions.

    View Recipe 05
